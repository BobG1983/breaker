# Text Input in Bevy 0.18.1

## No Built-in TextInput Component

Bevy 0.18.1 has NO `TextInput`, `TextInputPlugin`, or `ReceivedCharacter` types in bevy::input.
The keyboard module contains only: `KeyboardInput`, `KeyboardFocusLost`, `Key`, `KeyCode`,
`NativeKey`, `NativeKeyCode`, and `keyboard_input_system`.

`ReceivedCharacter` was deprecated/removed because it had ordering problems with `KeyboardInput`.

## The Canonical Pattern: KeyboardInput.text Field

Source: Official Bevy example `examples/input/text_input.rs` (verified from v0.18.0 tag)

```rust
use bevy::input::keyboard::{Key, KeyboardInput};
use bevy::prelude::*;

fn text_input_system(mut reader: MessageReader<KeyboardInput>, mut text: ResMut<MyTextField>) {
    for input in reader.read() {
        // Filter: only on press (not release)
        if !input.state.is_pressed() {
            continue;
        }
        // NOTE: The official example does NOT check input.repeat
        // OS key-repeat events DO come through with state.is_pressed() == true
        // To suppress repeats, check: if input.repeat { continue; }

        match (&input.logical_key, &input.text) {
            (Key::Backspace, _) => { text.0.pop(); }
            (_, Some(chars)) => {
                // Filter control characters (Escape etc. produce Some text on some platforms)
                if chars.chars().all(|c| !c.is_ascii_control()
                    && !('\u{e000}'..='\u{f8ff}').contains(&c))
                {
                    text.0.push_str(chars);
                }
            }
            _ => {}
        }
    }
}
```

## Key Field: KeyboardInput.text

- Type: `Option<SmolStr>`
- Contains the text character(s) produced by the key press
- `Some("1")` for digit key 1, `Some("a")` for letter a, etc.
- `None` for non-text keys (arrows, F-keys, modifier keys)
- On Windows with dead keys: may contain 2 chars (dead key char + current char)
- Matches `Key::Character(s)` variant of `logical_key` in most cases

## Key Field: KeyboardInput.repeat

- Type: `bool`
- `true` when this event was generated by OS key-repeat (key held down)
- The official Bevy example does NOT filter on this — it allows repeat character insertion
- For a seed field where you want repeats (e.g., holding backspace to clear), do NOT filter
- For single-press-only actions, add: `if input.repeat { continue; }`

## ButtonInput<KeyCode>::just_pressed vs KeyboardInput

- `just_pressed()`: fires exactly ONCE per physical key press (no OS repeat)
  - Returns `true` for exactly 1 frame after the physical key goes down
  - Does NOT repeat even if the key is held
- `KeyboardInput` messages: include both initial press AND OS key-repeat events
  - The `repeat: bool` field distinguishes initial press from held-key repeats
  - Use `repeat` field to filter if you want just_pressed behavior from messages

## Digit-Only Filtering

For a seed field accepting only 0-9:
```rust
(_, Some(chars)) => {
    let digits: String = chars.chars().filter(|c| c.is_ascii_digit()).collect();
    text.0.push_str(&digits);
}
```

Or match on `Key::Character`:
```rust
(Key::Character(s), _) if s.chars().all(|c| c.is_ascii_digit()) => {
    text.0.push_str(s);
}
```

## Focus Pattern (Custom Resource)

No built-in focus system for text fields. Pattern: use a `Resource` or component flag.

```rust
#[derive(Resource, Default)]
struct SeedFieldFocused(bool);

fn seed_input_system(
    focused: Res<SeedFieldFocused>,
    mut reader: MessageReader<KeyboardInput>,
    mut seed: ResMut<SeedField>,
) {
    if !focused.0 { return; }  // Only consume when focused
    for input in reader.read() {
        // ... handle input
    }
}
```

Toggling: click on the field entity triggers `SeedFieldFocused(true)`.
Clicking elsewhere or pressing Escape triggers `SeedFieldFocused(false)`.

## IME Support

Bevy 0.18 has `Ime` message (in `bevy::window`, derives `Message`):
- Variants: `Preedit`, `Commit`, `Enabled`, `Disabled`
- Requires `window.ime_enabled = true` to activate
- For digit-only input, IME is not needed and can be ignored entirely

## Clipboard / Paste

No clipboard API in Bevy 0.18.1 core. Not in bevy::input, not in bevy::window.
For paste support, would need an external crate (none currently well-established for 0.18).
For a seed field accepting only digits, paste is a nice-to-have, not essential.

## Third-Party: bevy_simple_text_input 0.14.1

- Supports `bevy ^0.18.0` (verified from docs.rs)
- Types: `TextInput` (marker), `TextInputPlugin`, `TextInputValue`, `TextInputSettings`,
  `TextInputInactive`, `TextInputPlaceholder`, `TextInputTextFont`, `TextInputTextColor`
- `TextInputInactive(bool)` — when `true`, ignores keyboard input and hides cursor
- Messages emitted: `TextInputSubmitMessage` (on Enter), `TextInputMaxLengthMessage`
- NO built-in digit filtering — must be implemented externally
- NO clipboard support
- Does NOT filter `repeat` events (processes all is_pressed events including OS repeats)
- Cursor movement supported: char left/right, word left/right, line start/end
- Would add a crate dependency; not in breaker-game's Cargo.toml
- Source: docs.rs/bevy_simple_text_input/0.14.1

## Recommendation for This Project

Roll a custom seed field using `MessageReader<KeyboardInput>` directly:
- No new crate dependency
- Filter to `state.is_pressed()` to skip releases
- Optionally filter `repeat` (for backspace, repeats are helpful; for digit entry, repeats are fine)
- Filter chars to `is_ascii_digit()` only
- Use a focused component/resource to suppress input when not focused
- Keep it in the `run` or `ui` domain as a simple resource + system
