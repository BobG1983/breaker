# GameConfig Rust Default vs RON File

When using `BoltConfig::default()` (or any `*Config` generated by the `GameConfig` derive macro) in tests, the values come from the **Rust `Default` impl on `*Defaults`**, NOT from the RON asset file.

## Example

`BoltDefaults::default().radius = 8.0` (Rust code in `bolt/resources.rs`)
`defaults.bolt.ron` has `radius: 14.0`

In tests, `BoltConfig::default().radius == 8.0`.

## Why This Matters for CCD Tests

Bolt radius affects `expanded_half` in `bolt_breaker_collision`:
```rust
let expanded_half = Vec2::new(half_w + r, half_h + r);
```

With `r = 8.0` (Rust default) and `half_w = 60.0`:
- Left AABB edge at `x = -68.0` (not -74.0 as the RON radius 14 would give)

Test bolt position must be outside this edge within `max_dist = speed * dt`:
- `max_dist` at speed 360.5, dt=1/64 ≈ 5.63
- Use `x = -(half_w + r + gap)` where `gap / cos(angle_from_x) < max_dist`

## How to Apply

Always verify which `Default` impl is used in tests. When setting up CCD test positions, compute `expanded_half` using `BoltConfig::default().radius` (Rust default), not the RON file value.
