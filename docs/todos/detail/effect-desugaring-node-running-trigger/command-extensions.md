# Effect Command Extensions — Behavioral Spec

Command extensions are the mechanism by which walk_effects and bridge systems defer effect execution and cross-entity mutation. They implement `EntityCommand` (Bevy 0.18) so they execute with exclusive `&mut World` access when Bevy flushes the command queue.

Bridge systems are regular Bevy systems with `Commands` parameter. walk_effects takes `&mut Commands` and calls these extensions for any operation that needs world access.

## Trait

```rust
pub(crate) trait EffectCommandsExt {
    fn fire_effect(&mut self, entity: Entity, effect: EffectType, source: SourceId, context: TriggerContext);
    fn reverse_effect(&mut self, entity: Entity, effect: ReversibleEffectType, source: SourceId);
    fn stamp_effect(&mut self, target: Entity, source: SourceId, tree: ValidTree);
    fn transfer_effect(&mut self, target: Entity, source: SourceId, tree: ValidTree);
    fn remove_source(&mut self, entity: Entity, source: SourceId);
    fn equip_chip(&mut self, entity: Entity, defs: &[ValidDef], source: SourceId);
    fn unequip_chip(&mut self, entity: Entity, source: SourceId);
}

impl<'w, 's> EffectCommandsExt for Commands<'w, 's> {
    // Each method queues an EntityCommand that executes during command flush
}
```

## fire_effect

**Purpose**: Execute an effect on an entity.

**Signature**: `fn fire_effect(&mut self, entity: Entity, effect: EffectType, source: SourceId, context: TriggerContext)`

**Behavior**:
1. Queue a `FireEffect` command
2. When command flushes (has `&mut World`):
   - Call `dispatch_fire(&effect, entity, &source, &context, world)`
   - dispatch_fire is a big match on EffectType — calls the per-effect module's fire() function
   - Each fire() function reads entity components from World and spawns/mutates as needed
   - If the effect spawns entities (Shockwave, ChainLightning, etc.), it does so via `world.spawn()`

**Called by**: walk_effects when encountering `Fire(effect)` in the tree, and by meta-effects (CircuitBreaker, EntropyEngine) firing sub-effects.

**Recursion**: If a fired effect triggers further dispatches (e.g., spawn → Added<T> bridge → more effects), the depth counter on TriggerContext prevents infinite loops (MAX_DISPATCH_DEPTH = 10).

```rust
struct FireEffect {
    effect: EffectType,
    source: SourceId,
    context: TriggerContext,
}

impl EntityCommand for FireEffect {
    fn apply(self, entity: Entity, world: &mut World) {
        dispatch_fire(&self.effect, entity, &self.source, &self.context, world);
    }
}
```

## reverse_effect

**Purpose**: Reverse a previously fired reversible effect on an entity.

**Signature**: `fn reverse_effect(&mut self, entity: Entity, effect: ReversibleEffectType, source: SourceId)`

**Behavior**:
1. Queue a `ReverseEffect` command
2. When command flushes:
   - Call `dispatch_reverse(&effect, entity, &source, world)`
   - dispatch_reverse is a big match on ReversibleEffectType — calls the per-effect module's reverse() function
   - reverse() undoes what fire() did (e.g., remove speed multiplier from ActiveSpeedBoosts)

**Called by**: walk_effects when encountering `Reverse(effect)` (generated by During/Until desugaring), and by condition monitors deactivating During entries.

```rust
struct ReverseEffect {
    effect: ReversibleEffectType,
    source: SourceId,
}

impl EntityCommand for ReverseEffect {
    fn apply(self, entity: Entity, world: &mut World) {
        dispatch_reverse(&self.effect, entity, &self.source, world);
    }
}
```

## stamp_effect

**Purpose**: Permanently add a tree to a target entity's BoundEffects. The entry persists and re-evaluates on each matching trigger.

**Signature**: `fn stamp_effect(&mut self, target: Entity, source: SourceId, tree: ValidTree)`

**Behavior**:
1. Queue a `StampEffect` command on the **target** entity (may be different from the entity being walked)
2. When command flushes:
   - Get target entity's `&mut BoundEffects` (insert default if absent)
   - Determine the key from the tree root: if `When(trigger, _)` → key is trigger; if `During(condition, _)` → key is condition; etc.
   - Push a `BoundEntry { key, source, tree }` into the appropriate Vec (triggers or conditions)
   - Update the source reverse index: `sources.entry(source).or_default().push(BoundKey::Trigger(index))`

**Called by**: walk_effects when encountering `On(target, Stamp(tree))` — the target entity is resolved from the TriggerContext participant.

```rust
struct StampEffect {
    source: SourceId,
    tree: ValidTree,
}

impl EntityCommand for StampEffect {
    fn apply(self, entity: Entity, world: &mut World) {
        let mut bound = world.get_mut::<BoundEffects>(entity)
            .expect("StampEffect target must have BoundEffects");
        // Insert entry, update reverse index
    }
}
```

## transfer_effect

**Purpose**: One-shot add a tree to a target entity's StagedEffects. The entry is consumed when its trigger fires.

**Signature**: `fn transfer_effect(&mut self, target: Entity, source: SourceId, tree: ValidTree)`

**Behavior**:
1. Queue a `TransferEffect` command on the **target** entity
2. When command flushes:
   - Get target entity's `&mut StagedEffects` (insert default if absent)
   - Determine the trigger key from the tree root
   - Push a `StagedEntry { trigger, source, tree }` into the entries Vec

**Called by**: walk_effects when encountering `On(target, Transfer(tree))`.

**Difference from Stamp**: Transfer goes to StagedEffects (consumed on trigger match). Stamp goes to BoundEffects (permanent, re-arms).

```rust
struct TransferEffect {
    source: SourceId,
    tree: ValidTree,
}

impl EntityCommand for TransferEffect {
    fn apply(self, entity: Entity, world: &mut World) {
        let mut staged = world.get_mut::<StagedEffects>(entity)
            .expect("TransferEffect target must have StagedEffects");
        // Insert entry
    }
}
```

## remove_source

**Purpose**: Remove all effect entries from a source (chip unequip cleanup).

**Signature**: `fn remove_source(&mut self, entity: Entity, source: SourceId)`

**Behavior**:
1. Queue a `RemoveSource` command
2. When command flushes:
   - Get `&mut BoundEffects`: look up source in reverse index, remove all entries at those indices, rebuild reverse index
   - Get `&mut StagedEffects`: retain only entries where `entry.source != source`
   - Get `&mut OnSpawnEffectRegistry` (resource): remove entries with matching source from all entity type Vecs

**Called by**: `unequip_chip` command extension, and by condition monitors cleaning up scope sources on deactivation.

## equip_chip

**Purpose**: Process a chip's ValidDefs and route them onto the appropriate entities.

**Signature**: `fn equip_chip(&mut self, entity: Entity, defs: &[ValidDef], source: SourceId)`

**Behavior**:
1. For each ValidDef:
   - Match on `def.route_target`:
     - `Bolt` → stamp tree onto primary bolt entity
     - `Breaker` → stamp tree onto breaker entity
     - `ActiveBolts` → stamp tree onto all existing bolt entities (point-in-time snapshot)
     - `EveryBolt` → stamp onto existing bolts (ActiveBolts) + register in OnSpawnEffectRegistry for future bolts
     - `PrimaryBolts` → stamp onto entities with PrimaryBolt marker
     - `ExtraBolts` → stamp onto entities with ExtraBolt marker
     - Same patterns for Cell/Wall/Breaker variants
   - Each stamp uses `stamp_effect` command internally

**Called by**: Chip domain at chip equip time.

**Note**: This is the integration point between the chip domain and the effect domain. The chip domain calls `commands.equip_chip(entity, &chip.effects, chip.name.clone())`.

## unequip_chip

**Purpose**: Remove all effects from a chip source.

**Signature**: `fn unequip_chip(&mut self, entity: Entity, source: SourceId)`

**Behavior**:
1. Call `remove_source(entity, source)` to clean BoundEffects/StagedEffects/OnSpawnEffectRegistry
2. Any effects already fired (spawned entities, active stat boosts) remain — detach on unequip (see decisions.md #12)

**Called by**: Chip domain at chip unequip time.

## dispatch_fire / dispatch_reverse

These are NOT command extensions — they're plain functions called inside command execution.

```rust
/// Match on EffectType, call per-effect module fire function.
fn dispatch_fire(effect: &EffectType, entity: Entity, source: &str, context: &TriggerContext, world: &mut World) {
    match effect {
        EffectType::SpeedBoost(v) => speed_boost::fire(entity, *v, source, world),
        EffectType::Shockwave(cfg) => shockwave::fire(entity, cfg, source, context, world),
        EffectType::Explode(cfg) => explode::fire(entity, cfg, source, context, world),
        EffectType::SpawnBolts(cfg) => spawn_bolts::fire(entity, cfg, source, world),
        // ... one arm per EffectType variant
    }
}

/// Match on ReversibleEffectType, call per-effect module reverse function.
fn dispatch_reverse(effect: &ReversibleEffectType, entity: Entity, source: &str, world: &mut World) {
    match effect {
        ReversibleEffectType::SpeedBoost(v) => speed_boost::reverse(entity, *v, source, world),
        // ... one arm per ReversibleEffectType variant
    }
}
```

CircuitBreaker and EntropyEngine call `dispatch_fire` directly for their sub-effects (incrementing the depth counter on the TriggerContext).
