# Tree Types

The effect system uses two parallel type trees: **Valid** (builder output, structurally enforced) and **Raw** (RON serde, permissive).

## ValidDef (definition-level)

Every top-level entry in a chip/breaker `effects: []` list produces a `ValidDef`. Route is required at root.

```rust
struct ValidDef {
    route_target: RouteTarget,
    tree: ValidTree,
}
```

## ValidTree (inner tree)

Lives inside Route, and inside Stamp/Transfer payloads.

```rust
enum ValidTree {
    Fire(EffectType),                               // fire on This (implicit)
    Sequence(Vec<ValidTree>),                       // ordered multi-fire
    When(Trigger, Box<ValidTree>),                  // gate — evaluate inner on trigger match
    Once(Trigger, Box<ValidTree>),                  // one-shot gate — self-removes after first match
    During(Condition, ValidScopedTree),             // state-scoped — fire on start, reverse on end
    Until(Trigger, ValidScopedTree),                // event-scoped — fire immediately, reverse on trigger
    Spawned(EntityType, Box<ValidTree>),            // fire on Added<T>
    On(ParticipantTarget, ValidTerminal),           // redirect to non-This target
}
```

## ValidScopedTree (During/Until inner)

Enforces reversibility for direct Fire. Relaxes to AnyFire for nested When.

```rust
enum ValidScopedTree {
    Fire(ReversibleEffectType),                     // direct fire — reversible only
    Sequence(Vec<ReversibleEffectType>),            // ordered multi-fire — all must be reversible
    When(Trigger, Box<ValidTree>),                  // nested When — relaxed (any effect OK)
    On(ParticipantTarget, ValidScopedTerminal),     // redirect — reversible only
}
```

## ValidTerminal (On target actions)

```rust
enum ValidTerminal {
    Fire(EffectType),                               // any effect, immediate
    Stamp(Box<ValidTree>),                          // permanent → target's BoundEffects
    Transfer(Box<ValidTree>),                       // one-shot → target's StagedEffects
    Reverse(ReversibleEffectType),                  // internal — generated by During/Until
}

enum ValidScopedTerminal {
    Fire(ReversibleEffectType),                     // reversible only
    Stamp(Box<ValidTree>),                          // stamp always OK
    Transfer(Box<ValidTree>),                       // transfer always OK
}
```

`Reverse` never appears in RON. It's generated internally when During/Until desugar their reversal entries.

## Raw Types (RON schema)

Permissive types for serde. Derive both `Serialize` + `Deserialize`.

```rust
struct RawDef {
    route: RawRouteTarget,
    tree: RawTree,
}

enum RawTree {
    Fire(EffectType),
    Sequence(Vec<RawTree>),
    When(Trigger, Box<RawTree>),
    Once(Trigger, Box<RawTree>),
    During(Condition, Box<RawTree>),
    Until(Trigger, Box<RawTree>),
    Spawned(EntityType, Box<RawTree>),
    On(RawParticipant, Box<RawTerminal>),
}

enum RawTerminal {
    Fire(EffectType),                   // validation checks reversibility
    Stamp(Box<RawTree>),
    Transfer(Box<RawTree>),
}

enum RawParticipant {
    BumpTarget(BumpTarget),
    ImpactTarget(ImpactTarget),
    DeathTarget(DeathTarget),
    BoltLostTarget(BoltLostTarget),
}
```

RON uses fully qualified participant names: `On(BumpTarget::Bolt, ...)`, `On(ImpactTarget::Impactee, ...)`.

## Loader (RON → Valid)

```
RON file → deserialize → RawDef → load_def() → Result<ValidDef, EffectError>
```

The loader walks the raw tree and validates:
- During/Until direct Fire is reversible (Error if not)
- On participant matches enclosing trigger context
- Route target is valid

## Builder (code → Valid)

```
EffectDef::route(target) → RouteContext → .when()/.during()/.fire() → ValidDef
EffectTree::when(trigger) → TriggerChain → .fire()/.on() → ValidTree
```

Typestate machine enforces constraints at compile time. See the builder-design todo detail for the full state diagram.

## Round-Trip

```
ValidDef → .to_raw() → RawDef → serialize → RON file
```

Enables content tooling to read and re-serialize effect definitions.
