# Pillar 9: Every Run Tells a Story

**The game is a unique situation generator, not a consumable experience.**

Every run must produce a sequence of moments that could only happen in that specific combination — the seed, the chip offerings, the player's choices, the mechanical execution, the near-misses and clutch clears. No two runs should feel the same, and the stories that emerge should be worth telling.

This is what Jason Rohrer calls a "unique situation generator" — a game whose systems produce novel, unrepeatable experiences through emergence rather than scripted content. The roguelite structure is the engine: seeded randomness determines the options, player skill determines the outcome, and the build-crafting system creates a narrative arc within each run (early struggle -> build comes online -> broken build fires on all cylinders OR collapses under pressure).

**Game span matters more than session length.** How long the game stays part of your life — days, weeks, months — determines whether you keep talking about it, streaming it, recommending it. A run that produces a unique story is a run that gets retold. A build that breaks the game in a way you've never seen before is a moment that gets shared. This sustained engagement is what drives word-of-mouth in a world where press coverage no longer moves the needle.

**The systems must be deep enough that novelty compounds.** With 3 chips, every run feels the same. With 20+ chips, rarity tiers, stacking, evolution recipes, and procedural node sequences, the combinatorial space explodes. The player who has done 100 runs should still encounter chip combinations and synergies they haven't seen before. The wiki should be full of interactions the developers didn't explicitly design — emergent behavior that rewards system mastery and creates discovery moments worth sharing.

**Design for stories, not balance sheets.** When choosing between a "balanced" design that produces consistent, predictable runs and a "spiky" design that occasionally produces wild outliers — choose spiky. The run where everything goes wrong is a story. The run where a rare chip drops on node 2 and the build comes online absurdly early is a story. The perfectly balanced run where nothing remarkable happens is forgettable.
